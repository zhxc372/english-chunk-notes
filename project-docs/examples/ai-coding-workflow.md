# AI Coding Workflow

## 主题目标

学习如何表达 AI 编程时代的任务拆分、代码审查、回滚、验证和工程风险。

## 主题文章

In the AI coding era, code generation has become much faster. A developer can ask an AI tool to create a feature, fix a bug, or write tests in a few minutes. However, faster output also creates a new problem. Human review has become the bottleneck.

A useful workflow should keep changes small and reviewable. Each task should have a clear scope, a simple acceptance criterion, and a safe recovery path. If a new version fails, the team should be able to roll back changes quickly.

AI can improve productivity, but it does not remove the need for engineering judgment. Developers still need to verify the result, test edge cases, and reduce security risks. In the long run, the best teams will not be the ones that generate the most code. They will be the ones that build reliable workflows around AI.
