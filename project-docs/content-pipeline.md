# 内容生产流水线

## 1. 总流程

```text
确定主题
→ 生成主题文章
→ 从文章抽取词块
→ 生成词块卡片 JSON
→ 人工审核
→ 生成音频
→ 放入网站
→ 用户收藏 / 闪卡训练
```

## 2. 主题文章要求

每篇文章：

- 150-250 词
- B1-B2 难度
- 句子尽量短
- 主题词块密度高
- 适合朗读
- 不写空泛套话
- 每段都能用于复述

不推荐：

- 800 词大作文
- 过度学术化
- 过度模板化
- 句子太长
- 内容太散

## 3. 词块抽取要求

优先提取：

- 2-4 词自然搭配
- 可写作表达
- 可听力识别表达
- 技术英语高频表达
- 考试主题表达

排除：

- 太简单
- 太生僻
- 不可迁移
- 只有一次性语境价值
- 过长术语串

## 4. AI 分工

AI 负责：

- 生成文章初稿
- 抽取候选词块
- 生成中文含义
- 生成搭配
- 生成例句
- 生成复述题
- 生成考试标签

人负责：

- 删除无价值词块
- 修改不自然例句
- 确认技术表达准确性
- 控制文章难度
- 控制主题边界

本地脚本负责：

- 保存 JSON
- 校验字段
- 去重
- 生成音频
- 导出 Anki

## 5. edge-tts 音频生成策略

第一版采用批量生成。

```text
lesson JSON
→ 读取 article.text 和 chunk.sentence
→ edge-tts 生成 mp3
→ 写入 public/audio/{lesson_id}/
→ 更新 JSON audio 路径
```

推荐音频文件名：

```text
article.mp3
chunk-001.mp3
chunk-002.mp3
chunk-003.mp3
```

## 6. 人工审核清单

每个 lesson 发布前检查：

- 文章是否自然、短、可复述？
- 词块是否值得背？
- 中文解释是否准确？
- 例句是否能听、能写、能复述？
- 技术句是否准确？
- 考试句是否不生硬？
- 音频路径是否存在？
- JSON 是否能被网站读取？
