# English Chunk Notes 项目宪法

## 0. 项目使命

本项目不是通用英语学习平台，而是一个 **考试英语 + 技术英语双轨主题词块文档站**。

目标是解决三个具体问题：

1. **写作中文味重**：通过主题文章、句型骨架、可套段落，把中文想法转成自然英文表达。
2. **词汇迷糊背不住**：通过主题词块、搭配、例句、闪卡，让词汇进入可复用语境。
3. **听力只能听单词**：通过文章音频、词块音频、听写、跟读、复述，让用户识别完整语块。

## 1. 不可违反的项目边界

第一版必须保持轻量。

必须坚持：

- 静态优先。
- 本地优先。
- 内容文件优先。
- AI 只作为内容生产工具，不作为网站运行时依赖。
- 网站不依赖数据库、不依赖登录、不依赖服务器。

第一版禁止：

- 用户注册 / 登录
- 云端数据库
- 云端同步
- 在线 AI 生成
- 在线 AI 批改
- 自动评分
- 复杂考试题库
- 社区功能
- 付费系统
- 大规模后台管理

## 2. 核心闭环

所有功能必须服务下面的闭环：

```text
主题文章
→ 文章听力音频
→ 从文章抽取主题词块
→ 词块含义 / 搭配 / 例句
→ 收藏
→ 闪卡
→ 听写 / 复述 / 写作套用
```

任何不能增强这个闭环的功能，第一版不做。

## 3. 内容原则

### 3.1 主题优先

词汇必须放在主题中学习。禁止建立孤立的大词表。

一个 lesson 必须包含：

- 主题说明
- 主题文章
- 文章音频
- 核心词块
- 词块例句
- 复述任务
- 写作套用任务

### 3.2 词块优先

优先收录 2-4 个词组成的自然表达。

优先收：

- support rollback
- human review
- review bottleneck
- define the scope
- test coverage
- security risk
- human oversight
- acceptance criteria

谨慎收：

- 单个普通词，如 good / important / thing / use / make
- 过长表达，如 microservice orchestration latency degradation
- 过于生僻、不可迁移的术语

### 3.3 每个词块必须可训练

每个词块至少包含：

- chunk
- meaning_cn
- collocations
- sentence
- sentence_cn
- rewrite_prompt_cn
- retelling_prompt
- tags
- exam_tags

没有训练任务的词块不能进入正式数据。

## 4. 考试适配原则

考试只是筛选入口，主题词块才是长期资产。

第一版只做考试标签，不做完整题库。

允许标签：

- IELTS
- TOEFL
- TOEIC
- PETS
- CET4
- CET6
- Postgraduate English
- Tech English

禁止第一版做：

- 完整雅思题库
- 完整托福听力题库
- 自动作文评分
- 自动口语评分
- 复杂考试报告

## 5. 技术英语原则

技术英语必须作为独立入口，但和考试英语共享底层词块结构。

第一批技术英语主题：

1. AI Coding Workflow
2. Software Engineering
3. Project Management
4. System Design
5. AI & Automation

技术英语词块必须有两种例句：

- technical_sentence：技术语境句
- exam_sentence：考试写作可迁移句

## 6. 听力原则

音频可以由 edge-tts 批量生成，但网站运行时只播放静态 MP3。

听力训练第一版只做：

- 播放文章音频
- 播放词块句音频
- 显示 / 隐藏文本
- 听写输入框
- 显示答案
- 标记听懂 / 没听懂

第一版不做：

- 发音评分
- 口音识别
- 在线 TTS
- 实时语音分析

## 7. 数据原则

内容必须落成本地 JSON / Markdown。

每个 lesson 必须可被版本控制、审查、修改和导出。

AI 生成的内容必须经过本地保存，不允许只存在聊天记录中。

## 8. 版权原则

可以借鉴公开学习网站的结构，不复制其正文、例句、题目和词汇解释。

可以借鉴 IELTS / TOEFL 等考试的常见主题分类，但所有文章、例句、解释、训练任务必须原创或由用户授权材料生成。

## 9. 工程原则

第一版技术栈：

```text
VitePress + Vue 组件 + JSON 数据 + localStorage + 静态音频
```

所有状态优先存 localStorage。

必须支持：

- 收藏词块
- 收藏主题
- 从收藏生成闪卡
- 导出收藏 JSON

必须避免：

- 过早引入状态管理框架
- 过早引入后端
- 过早引入复杂构建系统
- 过早引入数据库

## 10. 第一版成功标准

第一版成功只看 5 个问题：

1. 打开网站后，用户能不能 3 秒知道今天练什么？
2. 用户能不能按主题学习文章和词块？
3. 用户能不能收藏词块并生成闪卡？
4. 用户能不能听文章 / 词块句音频？
5. 用户能不能完成中译英、听写和复述任务？

这 5 个满足，第一版合格。
